  <action>
Create `app/services/playlist_engine.py` implementing the scoring pipeline from 04-RESEARCH.md:

1. **Constants**: FEATURE_WEIGHTS dict (energy=1.0, danceability=0.8, valence=0.7, tempo=0.5). ENERGY_NORM_DIVISOR = 0.3 (Essentia spectral_rms range). TEMPO_MIN = 40, TEMPO_RANGE = 180.

2. **score_track(track: Track, criteria: FeatureCriteria) -> float**: Weighted Euclidean distance scoring.
   - For each feature (energy, danceability, valence, tempo): if track has the value, compute distance to criteria midpoint, apply weight, apply 2x penalty if outside range.
   - Energy normalization: `min(track.energy / 0.3, 1.0)` before comparing to criteria (per research Pitfall 5).
   - Tempo normalization: `(track.tempo - 40) / 180` to [0,1].
   - Danceability and valence are already [0,1] from Essentia normalization.
   - If track has NO audio features (all None): use `metadata_feature_vector()` from audio_analyzer to get heuristic values, then score those. This implements D-11 metadata fallback.
   - If metadata fallback is used, add a small penalty (0.05) to prefer analyzed tracks.
   - Return `math.sqrt(distance / count)` where count = number of features scored.
   - If count == 0: return 1.0 (worst score).

3. **filter_candidates(session: Session, criteria: FeatureCriteria, track_count: int = 20, candidate_limit: int = 300) -> list[tuple[Track, float]]**: 
   - Query all tracks from SQLite.
   - If criteria.genres is non-empty, filter tracks whose genre matches any (case-insensitive substring match).
   - If criteria.exclude_genres is non-empty, exclude tracks whose genre matches any.
   - Score each remaining track with score_track().
   - Sort by score ascending (lower = better).
   - Return top `candidate_limit` tracks as (Track, score) tuples. The `track_count` is used by the caller (chat service) to tell the LLM how many to pick from these candidates.

4. **format_candidates_for_llm(candidates: list[tuple[Track, float]], limit: int = 300) -> str**: Compact pipe-delimited format for LLM context window efficiency (per research). Format: `ID|Title|Artist|Genre|Energy|Tempo|Dance|Valence` header, then one line per track with values truncated. "?" for missing features.
