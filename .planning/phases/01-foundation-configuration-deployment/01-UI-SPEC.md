# Phase 1 — UI Design Contract

> Visual and interaction contract for the Foundation, Configuration & Deployment phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (Jinja2 templates + Tailwind CSS v4 standalone CLI) |
| Preset | not applicable |
| Component library | none (hand-built HTML components via Jinja2 partials) |
| Icon library | Lucide Icons (SVG inlined via Jinja2 includes, vendored locally) |
| Font | Inter (self-hosted woff2, loaded from /static/fonts/) |

**Rationale:** Stack is Python/FastAPI with server-rendered Jinja2 templates, HTMX for dynamic interactions, and Alpine.js for micro-interactions. No React/Vue, so no shadcn. All frontend assets vendored locally for offline NAS operation (D-05 from CONTEXT.md anti-CDN pattern).

---

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding between icon and label |
| sm | 8px | Compact element spacing, input padding vertical |
| md | 16px | Default element spacing, input padding horizontal, form field gaps |
| lg | 24px | Section padding within cards, gap between form sections |
| xl | 32px | Card padding, layout gaps between service config blocks |
| 2xl | 48px | Page-level vertical padding (top/bottom of main content) |
| 3xl | 64px | Not used in Phase 1 |

Exceptions: Touch target minimum height of 44px for all interactive buttons (Test Connection, Save, Reconfigure). This is not a spacing token but a minimum `min-height` on clickable elements.

## Color

Plex-ecosystem dark theme. Colors derived from official Plex brand palette (#e5a00d accent, #282a2d surface) and extended for a complete dark UI.

| Role | Token | Value | Usage |
|------|-------|-------|-------|
| Dominant (60%) | `--color-surface-primary` | `#1f1f23` | Page background, app shell background |
| Secondary (30%) | `--color-surface-card` | `#2a2a2e` | Service config cards, nav bar background, form containers |
| Elevated | `--color-surface-elevated` | `#35353a` | Input fields, hover states on cards |
| Accent (10%) | `--color-accent` | `#e5a00d` | See reserved list below |
| Accent hover | `--color-accent-hover` | `#f0b429` | Hover state for accent elements |
| Text primary | `--color-text-primary` | `#e8e8ed` | Body text, headings, input values |
| Text secondary | `--color-text-secondary` | `#a1a1aa` | Descriptions, placeholders, timestamps |
| Text muted | `--color-text-muted` | `#63637a` | Disabled text, masked credential dots |
| Success | `--color-success` | `#22c55e` | Connection test success indicator, success toast |
| Error / Destructive | `--color-error` | `#ef4444` | Connection test failure, validation errors, destructive actions |
| Border | `--color-border` | `#3a3a42` | Card borders, input borders, dividers |
| Border focus | `--color-border-focus` | `#e5a00d` | Input focus ring (matches accent) |

**Accent reserved for:**
1. "Test Connection" button (primary CTA per service)
2. "Save Configuration" button
3. Navigation active state indicator
4. Input focus ring / border
5. Welcome page "Get Started" CTA

**Accent is NOT used for:** Body text links (use text-primary with underline), status indicators (use success/error), card backgrounds, borders at rest.

## Component Inventory (Phase 1)

These are the Jinja2 template partials needed for this phase:

| Component | Template Path | Interactions |
|-----------|---------------|-------------|
| App Shell (base layout) | `templates/base.html` | Static layout: nav bar + main content area |
| Nav Bar | `templates/partials/nav.html` | Horizontal top bar, gear icon link to /settings, app logo/name |
| Welcome Page | `templates/pages/welcome.html` | First-run landing; single CTA to begin setup |
| Settings Page | `templates/pages/settings.html` | Three service config sections stacked vertically |
| Service Config Card | `templates/partials/service_card.html` | Card container for each service (Plex, Ollama, Lidarr) |
| Connection Form | `templates/partials/{service}_form.html` | URL + credential inputs, "Test Connection" button; HTMX POST |
| Connection Status | `templates/partials/connection_status.html` | Success: green check + server name + dynamic options; Error: red X + message |
| Credential Mask | (inline in service_card) | Shows `********` with "Reconfigure" link when configured |
| Loading Spinner | `templates/partials/spinner.html` | HTMX indicator shown during connection test |

### Interaction Patterns

**Test-and-Configure (HTMX):**
1. User fills URL + token/key fields
2. Clicks "Test Connection" (accent-colored button)
3. HTMX sends `hx-post` to `/api/settings/{service}/test`
4. Spinner shows in `#status-{service}` via `hx-indicator`
5. Server returns HTML partial swapped into `#result-{service}` via `hx-target`
6. On success: partial shows green status + dynamic select (library/model/profile)
7. On failure: partial shows red status + error message + retry guidance
8. User selects option from dynamic dropdown, clicks "Save Configuration"

**Progressive Setup (D-03):**
1. First launch: redirect to `/settings` with welcome banner
2. Plex section is expanded and active; Ollama and Lidarr sections show "Configure Plex first" disabled state
3. After Plex is saved: Ollama and Lidarr sections become active (HTMX swap or page reload)
4. Ollama and Lidarr are independently optional -- each shows "Skip for now" option

**Credential Masking (D-04):**
1. After saving, the form fields are replaced with masked display: `URL: http://plex:32400` (visible) + `Token: ********` (masked)
2. A "Reconfigure" link replaces the masked view with the form (pre-filled URL, empty token)

## Layout

### App Shell
- Full-width page, max content width of 720px centered (settings is a single-column form page)
- Nav bar: full-width, height 56px, `surface-card` background, bottom border `border` color
- Nav bar content: app name left-aligned, settings gear icon right-aligned
- Main content: centered column with `2xl` (48px) top padding

### Settings Page
- Three service cards stacked vertically with `md` (16px) gap
- Each card: `surface-card` background, `border` border, 1px rounded corners (8px radius), `xl` (32px) padding
- Progressive disclosure: Plex card always visible; Ollama/Lidarr cards visible but conditionally enabled

### Welcome Page
- Centered vertically and horizontally in the viewport
- App name (Display size), description (Body size), single accent-colored CTA button
- No cards, no nav complexity -- clean entry point

## Accessibility Baseline

| Requirement | Implementation |
|-------------|---------------|
| Color contrast | All text meets WCAG AA (4.5:1 for body, 3:1 for large text). `#e8e8ed` on `#1f1f23` = 13.2:1. `#a1a1aa` on `#1f1f23` = 6.8:1. |
| Focus indicators | All interactive elements show `border-focus` (accent gold) 2px ring on `:focus-visible` |
| Form labels | All inputs have associated `<label>` elements with `for` attribute |
| Error identification | Error messages use `color-error` AND text description (not color alone) |
| Button states | Disabled buttons show `text-muted` color and `cursor-not-allowed` |
| Loading states | Spinner has `aria-label="Testing connection"` and button shows "Testing..." text |
