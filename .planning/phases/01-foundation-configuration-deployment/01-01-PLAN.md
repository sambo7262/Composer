<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User browser -> FastAPI | All user input for settings configuration |
| FastAPI -> SQLite | Credential storage at rest |
| FastAPI -> External services | Plex, Ollama, Lidarr connection tests (Plan 02) |
| Docker host -> Container | Volume mounts, port exposure |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-1-01 | Information Disclosure | settings_service.py | mitigate | ServiceConfigResponse NEVER includes encrypted_credential field; returns credential_set: bool only. get_decrypted_credential is internal-only function, not exposed via any API route. |
| T-1-02 | Information Disclosure | encryption.py | mitigate | Fernet key file stored in data volume with chmod 0o600; key never logged or returned in any response. |
| T-1-03 | Information Disclosure | docker-compose.yml | mitigate | No credential env vars in compose file (per D-10). All secrets stored in encrypted SQLite inside named volume. |
| T-1-04 | Information Disclosure | Dockerfile | mitigate | .dockerignore excludes .env, data/, tests/. No secrets baked into image layers. |
| T-1-05 | Tampering | /music volume | accept | Mounted read-only (:ro) per D-11/CONF-06. App cannot modify Plex media files. |
| T-1-06 | Denial of Service | api_health.py | accept | Single-user app on LAN. Health endpoint is lightweight. No rate limiting needed. |
</threat_model>
