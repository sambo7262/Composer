    1. Create app/services/analysis_service.py (per D-01, D-02, D-03, D-04, D-11, D-13, D-15, D-16, AUDIO-02, AUDIO-03, AUDIO-04):

       a. AnalysisStateEnum(str, Enum): IDLE, RUNNING, PAUSED, COMPLETED, FAILED

       b. AnalysisStatus dataclass (module-level singleton like SyncStatus):
          - state: AnalysisStateEnum = IDLE
          - total_tracks, analyzed_tracks, failed_tracks: int = 0
          - current_track: str = ""
          - avg_seconds_per_track: float = 0.0
          - errors: list[dict] = field(default_factory=list) -- last 50 errors max
          - _times: list[float] for rolling average (window=50)

          Property `eta_display` -> str: "~Xh Ym remaining" / "~Xm remaining" / "~Xs remaining"
          Based on (total - analyzed - failed) * avg_seconds_per_track

       c. `_detect_plex_music_root(session) -> str`:
          - Query first 10 tracks with file_path not null
          - Find longest common prefix of their file paths (os.path.commonpath)
          - If fewer than 2 tracks have file_path, check settings for a configured plex_music_root
          - Return the detected root or "/" as fallback

       d. `async def run_analysis() -> None`:
          - Guard: if state == RUNNING, return immediately (prevent concurrent)
          - Set state = RUNNING
          - Detect plex_music_root via _detect_plex_music_root
          - Query un-analyzed tracks: `SELECT * FROM track WHERE analyzed_at IS NULL AND file_path IS NOT NULL`
          - Set total_tracks = count of results
          - Loop over tracks:
            - Check if state changed to PAUSED -- if so, break (D-03)
            - Set current_track = f"{track.artist} - {track.title}"
            - Remap file path via remap_plex_path(track.file_path, plex_music_root)
            - Check if remapped file exists (os.path.isfile) -- skip if not, log warning
            - Call `await asyncio.to_thread(extract_features, remapped_path)` (CPU-bound, must not block event loop)
            - On success: update Track with all feature values + analyzed_at = now, clear analysis_error
            - On failure: set analysis_error = str(exception), increment failed_tracks, add to errors list (capped at 50)
            - Record elapsed time, update avg_seconds_per_track (rolling window of 50)
            - Commit each track individually (incremental progress survives crashes)
          - After loop: set state = COMPLETED (or PAUSED if stopped)

       e. `async def stop_analysis() -> None`:
          - Set state = PAUSED (D-03)
          - The run loop checks this flag each iteration and breaks

       f. `def get_analysis_status() -> AnalysisStatus`:
          - Return the module-level singleton

       g. Auto-trigger hook: `async def trigger_post_sync_analysis() -> None`:
          - Called after sync completes (D-01)
          - Check if state is not RUNNING and not PAUSED
          - If there are un-analyzed tracks with file_path, launch `asyncio.create_task(run_analysis())`

| Boundary | Description |
|----------|-------------|
| API client -> analysis endpoints | HTTP requests control start/stop of CPU-intensive background job |
| Analysis service -> filesystem | Reads audio files from /music mount based on Plex-reported paths |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-04 | Denial of Service | POST /api/analysis/start | mitigate | Prevent concurrent analysis runs -- guard in run_analysis rejects if already RUNNING. Rate limiting not needed (single-user app). |
| T-03-05 | Denial of Service | extract_features per track | mitigate | Skip files > 100MB via os.path.getsize check before analysis. Set 30-second implicit timeout by monitoring elapsed time. |
| T-03-06 | Information Disclosure | GET /api/analysis/status errors list | mitigate | Return only track title + generic error message in API responses. Full file paths and stack traces logged server-side only (T-03-03 from Plan 01). |
| T-03-07 | Tampering | sync auto-trigger | accept | Low risk -- trigger_post_sync_analysis only fires after successful sync, same trust level as manual trigger. |
</threat_model>
