       ```python
       from dataclasses import dataclass
       from enum import Enum

       class SyncStateEnum(str, Enum):
           IDLE = "idle"
           RUNNING = "running"
           COMPLETED = "completed"
           FAILED = "failed"

       @dataclass
       class SyncStatus:
           state: SyncStateEnum = SyncStateEnum.IDLE
           total_tracks: int = 0
           synced_tracks: int = 0
           last_synced: str | None = None
           error: str | None = None
       ```

    2. **Module-level singleton** `_sync_status = SyncStatus()` and `get_sync_status() -> SyncStatus` getter.

    3. **`async def run_sync()`** main entry point:
       - Check `_sync_status.state == RUNNING` -- if so, return immediately (prevent concurrent syncs)
       - Set state to RUNNING, reset counters
       - Get Plex credentials from settings_service: `get_decrypted_credential(session, "plex")`, `get_setting(session, "plex")` for url and extra_config.library_id
       - Check SyncState table for `last_sync_completed`:
         - If exists: call delta sync via `get_tracks_since()` first. If delta returns tracks, upsert them. If delta returns 0 but library has tracks, fall back to full sync.
         - If not exists: full sync
       - **Full sync loop** (per SYNC-01): Loop with `container_start` incrementing by `batch_size=200`. Call `get_library_tracks(url, token, library_id, container_start, 200)`. After each batch, call `_upsert_tracks(batch)` and update `_sync_status.synced_tracks`. Break when `len(batch) < batch_size` or `offset >= total`.
       - On success: set state to COMPLETED, update `_sync_status.last_synced` to ISO timestamp, update SyncState table `last_sync_completed`
       - On exception: set state to FAILED, store error string (never include the Plex token in error messages -- sanitize)
       - Use `asyncio.to_thread` for all DB writes (session operations) to avoid blocking event loop

    4. **`async def _upsert_tracks(track_dicts: list[dict])`** (per SYNC-01):

       - Commit in a single transaction per batch (200 tracks at a time to minimize WAL contention)

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| App -> Plex API | Plex token used to authenticate; PlexAPI returns untrusted track metadata |
| User -> Sync API | User triggers sync; must not be able to trigger concurrent syncs |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | Track metadata from Plex | accept | Plex is a trusted local service on the user's own network; metadata stored as-is |
| T-02-02 | Information Disclosure | Plex token in error messages | mitigate | sync_service.py sanitizes error messages: never include token in _sync_status.error |
| T-02-03 | Denial of Service | Concurrent sync triggers | mitigate | run_sync() checks _sync_status.state == RUNNING and rejects if already running |
| T-02-04 | Tampering | SQL injection via track metadata | mitigate | SQLModel ORM uses parameterized queries for all inserts/updates -- no raw SQL |
</threat_model>
