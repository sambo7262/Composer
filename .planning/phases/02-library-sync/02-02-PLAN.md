    1. Create `app/routers/api_library.py` with one endpoint (per D-08, D-09):

       **GET `/api/library/tracks`**:
       - Query params: `page: int = 1`, `per_page: int = 50`, `search: str = ""`, `sort: str = "title"`, `order: str = "asc"`
       - Validate `sort` against allowed columns: `["title", "artist", "album", "genre", "year"]`. Default to "title" if invalid.
       - Validate `order`: must be "asc" or "desc". Default to "asc" if invalid.
       - Build query: `select(Track)`
       - If `search` is non-empty: apply `or_()` filter with `Track.title.ilike(f"%{search}%")`, `Track.artist.ilike(f"%{search}%")`, `Track.album.ilike(f"%{search}%")`
       - Apply sorting: `getattr(Track, sort, Track.title)` with `.asc()` or `.desc()` based on `order`
       - Count total results: `select(func.count()).select_from(query.subquery())`
       - Apply pagination: `.offset((page - 1) * per_page).limit(per_page)`
       - Calculate pagination metadata: `total_pages = ceil(total / per_page)`, `has_prev = page > 1`, `has_next = page < total_pages`
       - Check `HX-Request` header to determine if HTMX request
       - If HTMX request: return `track_table.html` partial only
       - If full page request: return full library page (redirect to /library or return library.html with track data embedded)
       - Pass to template: `tracks`, `page`, `per_page`, `total`, `total_pages`, `has_prev`, `has_next`, `search`, `sort`, `order`

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User browser -> Library API | Search, sort, page params are untrusted user input |
| User browser -> Sync API | Sync trigger from UI |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-05 | Tampering | Search query param in /api/library/tracks | mitigate | SQLModel `ilike()` uses parameterized queries -- no raw SQL. Validate sort column against allowlist ["title", "artist", "album", "genre", "year"]. Validate order against ["asc", "desc"]. |
| T-02-06 | Denial of Service | Rapid sync trigger abuse | mitigate | run_sync() rejects if already RUNNING; api_sync returns existing progress banner |
| T-02-07 | Tampering | Page/per_page params (e.g., per_page=999999) | mitigate | Cap per_page at 100 max in api_library endpoint; validate page >= 1 |
| T-02-08 | Information Disclosure | Track data exposure | accept | Track metadata (title, artist, album) is not sensitive -- single-user app on local network |
</threat_model>
